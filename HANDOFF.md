# Osten-Sacken — HANDOFF / контекст проекту

> Документ для швидкого входу в контекст (новий чат / інший розробник).
> Оновлено: 2026-07-07.

## 1. Що це за проект

**Osten-Sacken** — односторінковий сайт-магазин (лендінг) для замовлення **титанових глушників під замовлення**. Мова інтерфейсу — **українська**. Аудиторія — власники нарізної зброї.

- Домен: **ostensacken.com** (куплений).
- Хостинг: **VPS на Ubuntu** (у процесі придбання). Локально сервер НЕ запускається (див. §6).
- Задум: основний сайт + **піддомен `admin.ostensacken.com`** з входом за паролем — повноцінна адмін-панель для керування контентом і замовленнями.
- При замовленні продавцю має падати **красивий HTML-лист** (Resend), можливо дубль у Telegram.

Замовник — не розробник. Потрібні прості інструкції та висока точність відповідності дизайну.

## 2. Поточний стан (етап)

**Етап: фронтенд лендінга, доводимо до 1:1 з макетом Figma.** Backend / адмінка / деплой — ще НЕ розпочаті.

Лендінг зібрано з нуля (не сирий Figma-експорт), адаптивний, з анімаціями. Секції готові й максимально зведені під макет. Тривають ітеративні правки точності за зауваженнями замовника.

## 3. Структура файлів

```
ostensacken.com/

├─ figma-to-html.zip         ← початковий сирий експорт з Figma (довідково)
├─ design/                   ← розпакований експорт (довідково, НЕ використовуємо напряму)
├─ public/                   ← ЛЕНДІНГ (статика)
│  ├─ index.html             ← вся розмітка лендінга
│  ├─ css/
│  │  ├─ fonts.css           ← @font-face (Squares Bold + Google Sans)
│  │  └─ site.css            ← всі стилі
│  ├─ js/site.js             ← вся поведінка (reveal, лічильник, каруселі, барабан, форма)
│  ├─ fonts/                 ← SquaresBold.otf, GoogleSans-Regular/Medium/Bold.ttf
│  └─ images/                ← всі ассети (див. §5)
├─ admin/                    ← ПОКИ ПОРОЖНЯ (майбутня адмінка: css/ js/)
├─ data/                     ← ПОКИ ПОРОЖНЯ (майбутні orders.json / content.json)
└─ scripts/
   ├─ static-preview.ps1     ← локальний превʼю-сервер (PowerShell HttpListener)
   └─ static-preview.js      ← той самий, але для Node (локально Node немає)
```

Референс-проект того ж замовника: `../obyava-mailer` — той самий стек (Express + Resend + вхід за паролем + Docker/PM2 деплой). Брати за зразок для backend/адмінки/деплою.

## 4. Дизайн-система (токени з Figma)

- Фон `--bg #FAF9F6` · текст/CTA/футер `--ink #010203` · вторинний текст `--muted #434343`
- Картки/шапка-таблиці `--panel #F5F5F5` · білий `#FFFFFF`
- Бордери: `--line #E5E5E5`, `--line-strong #A3A3A3`
- **АКЦЕНТ ЧЕРВОНИЙ `--accent #E31919`** (primary-кнопки, акцентне слово в заголовках, активні стани, аккордеон)
- Інфо-плашка: `--info-bg #DAE3FF`, `--info-txt #2719E3`
- Шрифти: **Squares Bold** (заголовки, кнопки; це «квадратний» дисплейний, фактично капсовий) + **Google Sans** (текст). Файли вже в `public/fonts/`, підключені у `fonts.css`.

**Стильові правила:** гострі кути (без скруглень); заголовки секцій UPPERCASE з одним словом червоним (`ВИБЕРІТЬ ТИП ГЛУШНИКА`); primary-кнопка = червона заливка + кремовий текст, при hover **подовжується + виїжджає іконка-приціл**.

### Як тягнути нові ассети з Figma
Підключено офіційний **Figma Dev Mode MCP** (`figma-desktop`, `http://127.0.0.1:3845/mcp`) — працює ЛИШЕ поки відкритий десктопний Figma під акаунтом замовника. Інструменти deferred (вантажити через ToolSearch): `get_design_context`, `get_screenshot`, `get_metadata`, `get_variable_defs`.
`get_design_context` повертає URL ассетів виду `http://localhost:3845/assets/<hash>.png|svg` — качати через PowerShell `Invoke-WebRequest` у `public/images`.
Головний фрейм макета: node-id `167-2320`. Компонент анімації карток калібру: `27-132`.

Вирізання фону (для темних секцій) робимо самі через `System.Drawing` LockBits у PowerShell (у ассетів Figma фон запечений). Поріг ~170, насиченість <26.

## 6. Локальне середовище (ВАЖЛИВО)

- На машині **НЕМАЄ Node.js і Python** (і `claude` CLI не в PATH). Тому реальний Express-сервер запускається лише на VPS.
- Локальне превʼю: **PowerShell HttpListener** — `scripts/static-preview.ps1`, порт 5173. Конфіг превʼю: `../.claude/launch.json` (runtimeExecutable `powershell`).
- **Скріншот-сервіс превʼю нестабільний** (часто зависає, особливо після resize). Обхід: перезапуск сервера дає 1 робочий кадр; перевіряти стилі через `preview_inspect`/`preview_eval` (вони працюють надійно).

## 7. Що зроблено (детально)

Лендінг `public/index.html`, секції зверху вниз:
1. **Header** — лого-SVG, навігація (Глушники · Калібри · Вартість · Як замовити? · Відгуки · Контакти), CTA Secondary (червона рамка).
2. **Hero** — заголовок «ТИТАНОВІ ГЛУШНИКИ ПІД ЗАМОВЛЕННЯ» (ПІД ЗАМОВЛЕННЯ червоним), фото-виріз прибите до правого краю сторінки, статичне.
3. **Stats** — Для всіх / 5500+ (лічильник) / 10+ років.
4. **Конфігуратор** «Виберіть ТИП глушника» — 2 картки (характеристики, тумблер Так/Ні, інфо-плашка, кнопка ЗАМОВИТИ зліва), фото в тонкій рамці, карусель (стрілка + крапки + слайди-болванки).
5. **Калібри** «Сумісні … КАЛІБРАМИ» — 5 карток. Desktop: hover → картка розширюється + зброя довертається на 90° у горизонталь + червона H2 + кнопка (за Figma 27:132), фікс. висота (немає стрибків). Mobile: вертикальний «барабан», центральна чітка + кнопка, крайні розмиті, зброя горизонтальна й **статична**.
6. **Таблиця цін** — .22 LR / .223 Rem / .30 Cal / .338, стандартний/інтегрований.
7. **«3 простих кроки»** (темна) — великий заголовок зверху-зліва в блоці, зліва велика гвинтівка під нахилом (вилазить знизу блоку), справа нумеровані кроки (білі лінії між цифрами) + кнопка.
8. **«Про нас»** (темна) — мішень-приціл + солдат + градієнт, `OSTEN - SACKEN` капсом, опис, контакти.
9. **«Відгуки клієнтів»** — карусель (плейсхолдери, треба реальні фото/відео).
10. **Форма** «Залишились ПИТАННЯ?» — рівно 4 поля з макета: Ім'я*, Телефон, Пошта*, Ваше питання* + кнопка ВІДПРАВИТИ + стани помилок. Кнопки з конфігуратора/калібрів підставляють дані у «Ваше питання».
11. **FAQ** «Часті ПИТАННЯ» — аккордеон (5), червона іконка +/×.
12. **Footer** — лого, контакти, навігація, копірайт + cutout-гвинтівка в куті.

Анімації: reveal-on-scroll, лічильник, hover-стани (лише `@media (hover:hover)` для карток калібру), каруселі, мобільний барабан, приціл на кнопках.
